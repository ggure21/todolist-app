## 1. 화면 구성 개요

### 1.1 페이지 목록

| 경로        | 페이지명         | 접근 조건 | 관련 UC              |
|-------------|------------------|-----------|----------------------|
| `/login`    | 로그인           | 비인증    | UC-02                |
| `/register` | 회원가입         | 비인증    | UC-01                |
| `/`         | 할일 목록 (메인) | 인증 필요 | UC-04~09             |
| `/profile`  | 프로필 (개인정보 수정) | 인증 필요 | UC-03           |

### 1.2 네비게이션 흐름

```
[비인증 사용자]
      │
      ├─── /register (회원가입) ──── 가입 성공 ───► /login
      │
      └─── /login (로그인) ────────── 로그인 성공 ─► / (할일 목록)
                                                         │
                                          ┌──────────────┤
                                          │              │
                                    /profile         / (메인)
                                    (프로필)     할일 CRUD
                                                 카테고리 추가
                                                 필터링
```

## 2. 공통 컴포넌트

### 2.1 헤더 (인증 후 공통)

```
┌─────────────────────────────────────────────────────────────────────┐
│  TodoListApp                              안녕하세요, 김지수님  [로그아웃] │
└─────────────────────────────────────────────────────────────────────┘
```

- 로고/서비스명 (좌측): 클릭 시 `/` 이동
- 사용자 이름 표시 (우측): User.name 값
- 로그아웃 버튼: 클릭 시 Zustand authStore 초기화 후 `/login` 이동

### 2.2 사이드바 (Desktop 전용 — 768px 초과)

```
┌────────────────────┐
│ 카테고리             │
│ ────────────────── │
│ ▶ 전체              │  ← 활성 상태
│   개인              │  ← 기본 카테고리 (수정/삭제 없음, BR-C-01)
│   업무              │
│   쇼핑              │
│ ────────────────── │
│   가정          [x] │  ← 사용자 정의 카테고리 (삭제 버튼 표시)
│   프로젝트      [x] │
│ ────────────────── │
│ [+ 카테고리 추가]    │
└────────────────────┘
```

- 기본 카테고리 (개인/업무/쇼핑): 삭제/수정 버튼 없음 (BR-C-01)
- 사용자 정의 카테고리: `[x]` 삭제 버튼 표시
- `[+ 카테고리 추가]`: 인라인 입력 폼 또는 작은 모달 트리거

### 2.3 카테고리 추가 인라인 폼 (사이드바 내)

```
┌────────────────────┐
│ [새 카테고리명____] │
│ [추가]  [취소]      │
│ ⚠ 이미 존재하는 이름입니다  │  ← 중복 시 표시 (BR-C-04)
└────────────────────┘
```

### 2.4 빈 상태 (EmptyState)

```
┌──────────────────────────────────────────────┐
│                                              │
│           할일이 없습니다                     │
│                                              │
│   필터 조건을 변경하거나 새 할일을 추가하세요  │
│                                              │
│              [ + 새 할일 추가 ]               │
│                                              │
└──────────────────────────────────────────────┘
```

### 5.4 상태 및 규칙

| 상태              | UI 처리                                                        |
|-------------------|----------------------------------------------------------------|
| 기본 상태         | 전체 할일 목록 표시 (카테고리: 전체, 기간초과: 전체, 완료: 전체) |
| 완료 항목         | 체크박스 `[x]`, 제목 취소선, 카테고리/날짜도 취소선 표시 (BR-T-04) |
| 기간 초과 항목    | `⚠ YYYY-MM-DD (초과)` 강조 표시 (BR-F-02)                     |
| 필터 0건          | 빈 상태 안내 문구 + 새 할일 추가 버튼 표시                     |
| 로딩 상태         | 목록 영역에 로딩 스피너 표시                                   |

**비즈니스 규칙 적용:**
- BR-C-01: 기본 카테고리(개인/업무/쇼핑)에는 수정/삭제 버튼 없음
- BR-C-04: 카테고리 추가 시 동일 사용자 내 이름 중복 불가
- BR-F-01~04: 카테고리, 기간초과, 완료여부 필터 AND 조건 동시 적용
