# TodoListApp 도메인 정의서

- 버전: 1.0.0
- 작성일: 2026-05-12
- 작성자: Business Analyst

---

## 1. 프로젝트 개요

### 1.1 문제 정의

| # | 문제 | 영향 |
|---|------|------|
| 1 | 개인별 할일을 체계적으로 관리할 수 있는 전용 앱 부재 | 할일 누락, 일정 관리 비효율 |
| 2 | 데이터베이스 기반의 사용자 인증 연동 할일 저장 기능 부재 | 데이터 유실, 멀티 디바이스 접근 불가 |
| 3 | 카테고리 단위 할일 분류 및 필터링 기능 부재 | 할일 혼재, 우선순위 파악 어려움 |

### 1.2 목적

인증된 사용자가 자신만의 할일을 카테고리별로 등록, 조회, 관리할 수 있는 개인 맞춤형 할일 관리 서비스를 제공한다.

### 1.3 서비스 범위

- 사용자 회원가입 및 인증
- 인증된 사용자의 개인 정보 관리
- 할일(Todo) 등록, 조회, 수정, 삭제
- 카테고리 관리 (기본 카테고리 + 사용자 정의 카테고리)
- 할일 목록 필터링 (카테고리, 기간, 완료 여부)

## 2. 도메인 용어 정의 (Ubiquitous Language)

| 용어 | 영문 | 정의 |
|------|------|------|
| 사용자 | User | 회원가입 후 인증을 통해 서비스를 이용하는 주체 |
| 인증 | Authentication | 등록된 자격증명(이메일, 비밀번호)으로 신원을 확인하는 행위 |
| 회원가입 | Registration | 서비스 이용을 위해 사용자 정보를 등록하는 행위 |
| 할일 | Todo | 사용자가 완료해야 할 특정 작업 단위 |
| 카테고리 | Category | 할일을 분류하는 그룹 단위. 기본 카테고리와 사용자 정의 카테고리로 구분 |
| 기본 카테고리 | Default Category | 시스템이 사전 제공하는 카테고리 (예: 개인, 업무, 쇼핑 등) |
| 사용자 정의 카테고리 | Custom Category | 사용자가 직접 생성한 카테고리 |
| 종료예정일 | Due Date | 할일을 완료해야 하는 목표 날짜 |
| 완료 여부 | Completion Status | 할일의 완료(Done) 또는 미완료(Pending) 상태 |
| 기간 종료 여부 | Overdue Status | 현재 날짜 기준 종료예정일 초과 여부 |
| 필터링 | Filtering | 특정 조건(카테고리, 기간, 완료 여부)으로 할일 목록을 좁혀 조회하는 기능 |

## 3. 핵심 도메인 객체

### 3.1 엔티티 (Entity)

#### User (사용자)

사용자 식별의 핵심 엔티티. 모든 할일과 카테고리의 소유 주체.

| 속성 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | UUID | PK, Not Null | 사용자 고유 식별자 |
| email | String | Unique, Not Null | 로그인 및 식별에 사용하는 이메일 |
| password | String | Not Null | 해시 처리된 비밀번호 |
| name | String | Not Null | 사용자 표시 이름 |
| created_at | DateTime | Not Null | 가입 일시 |
| updated_at | DateTime | Not Null | 정보 수정 일시 |

#### Todo (할일)

서비스의 핵심 엔티티. 특정 사용자에 귀속되며 카테고리로 분류된다.

| 속성 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | UUID | PK, Not Null | 할일 고유 식별자 |
| user_id | UUID | FK → User, Not Null | 소유 사용자 |
| category_id | UUID | FK → Category, Not Null | 분류 카테고리 |
| title | String | Not Null | 할일 제목 |
| description | String | Nullable | 할일 상세 설명 |
| due_date | Date | Nullable | 종료예정일 |
| is_completed | Boolean | Not Null, Default: false | 완료 여부 |
| completed_at | DateTime | Nullable | 완료 처리 일시 |
| created_at | DateTime | Not Null | 등록 일시 |
| updated_at | DateTime | Not Null | 수정 일시 |

#### Category (카테고리)

할일을 분류하는 엔티티. 기본 카테고리는 시스템 소유, 사용자 정의 카테고리는 특정 사용자 소유.

| 속성 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | UUID | PK, Not Null | 카테고리 고유 식별자 |
| user_id | UUID | FK → User, Nullable | 소유 사용자 (Null이면 기본 카테고리) |
| name | String | Not Null | 카테고리 이름 |
| is_default | Boolean | Not Null, Default: false | 기본 카테고리 여부 |
| created_at | DateTime | Not Null | 생성 일시 |

### 3.2 값 객체 (Value Object)

| 값 객체 | 설명 | 구성 요소 |
|---------|------|-----------|
| Credentials | 인증 자격증명 | email, password |
| DueDate | 종료예정일 및 초과 여부 | date, is_overdue (파생값) |
| CompletionStatus | 완료 상태 | is_completed, completed_at |

## 4. 비즈니스 규칙

### 4.1 사용자 규칙

| ID | 규칙 |
|----|------|
| BR-U-01 | 이메일은 전체 시스템에서 고유해야 하며 중복 가입이 불가하다. |
| BR-U-02 | 비밀번호는 저장 시 반드시 해시 처리되어야 하며 평문으로 저장할 수 없다. |
| BR-U-03 | 인증되지 않은 사용자는 할일 및 카테고리 기능에 접근할 수 없다. |
| BR-U-04 | 사용자는 자신의 개인 정보(이름, 비밀번호)만 수정할 수 있다. 이메일 변경은 허용하지 않는다. |

### 4.2 할일 규칙

| ID | 규칙 |
|----|------|
| BR-T-01 | 할일은 반드시 하나의 카테고리에 속해야 한다. |
| BR-T-02 | 할일의 제목(title)은 필수 입력값이다. |
| BR-T-03 | 사용자는 자신이 등록한 할일만 조회, 수정, 삭제할 수 있다. |
| BR-T-04 | 할일 완료 처리 시 completed_at 일시가 자동으로 기록된다. |
| BR-T-05 | 완료된 할일을 다시 미완료로 되돌릴 수 있으며, 이 경우 completed_at은 초기화된다. |
| BR-T-06 | 종료예정일(due_date)은 필수가 아니나, 입력된 경우 현재 날짜 이후여야 한다. (수정 시 예외 없음) |

### 4.3 카테고리 규칙

| ID | 규칙 |
|----|------|
| BR-C-01 | 기본 카테고리는 시스템이 제공하며 사용자가 수정하거나 삭제할 수 없다. |
| BR-C-02 | 사용자 정의 카테고리는 해당 사용자만 조회, 수정, 삭제할 수 있다. |
| BR-C-03 | 카테고리 삭제 시 해당 카테고리에 속한 할일이 존재하면 삭제할 수 없다. |
| BR-C-04 | 동일 사용자 내에서 카테고리 이름은 중복될 수 없다. |

### 4.4 필터링 규칙

| ID | 규칙 |
|----|------|
| BR-F-01 | 카테고리 필터: 선택한 카테고리에 속한 할일만 표시한다. |
| BR-F-02 | 기간 종료 여부 필터: 현재 날짜(today) 기준으로 due_date가 초과된 할일을 구분하여 표시한다. |
| BR-F-03 | 완료 여부 필터: is_completed 값 기준으로 완료/미완료 할일을 구분하여 표시한다. |
| BR-F-04 | 복수의 필터 조건은 AND 조건으로 동시에 적용될 수 있다. |

## 5. 주요 유스케이스

### UC-01: 회원가입

- 액터: 비인증 사용자
- 사전 조건: 해당 이메일로 가입된 계정이 없어야 한다.
- 기본 흐름:
  1. 사용자가 이메일, 비밀번호, 이름을 입력한다.
  2. 시스템이 이메일 중복 여부를 검증한다.
  3. 시스템이 비밀번호를 해시 처리하여 사용자 정보를 저장한다.
  4. 회원가입이 완료되고 로그인 화면으로 이동한다.
- 예외 흐름: 이메일 중복 시 오류 메시지를 반환하고 가입을 중단한다.

### UC-02: 로그인 (사용자 인증)

- 액터: 비인증 사용자
- 사전 조건: 등록된 계정이 존재해야 한다.
- 기본 흐름:
  1. 사용자가 이메일과 비밀번호를 입력한다.
  2. 시스템이 이메일로 사용자를 조회하고 비밀번호 해시를 검증한다.
  3. 검증 성공 시 인증 토큰을 발급하고 메인 화면으로 이동한다.
- 예외 흐름: 이메일 미존재 또는 비밀번호 불일치 시 단일 오류 메시지를 반환한다. (보안상 구분하지 않음)

### UC-03: 개인 정보 수정

- 액터: 인증된 사용자
- 사전 조건: 사용자가 로그인 상태여야 한다.
- 기본 흐름:
  1. 사용자가 수정할 이름 또는 비밀번호를 입력한다.
  2. 비밀번호 변경 시 현재 비밀번호를 재확인한다.
  3. 시스템이 변경 사항을 저장하고 완료를 알린다.
- 예외 흐름: 현재 비밀번호 불일치 시 변경을 거부한다.

### UC-04: 카테고리 추가

- 액터: 인증된 사용자
- 사전 조건: 사용자가 로그인 상태여야 한다.
- 기본 흐름:
  1. 사용자가 카테고리 이름을 입력한다.
  2. 시스템이 동일 사용자 내 카테고리 이름 중복 여부를 검증한다.
  3. 시스템이 사용자 정의 카테고리를 생성하여 저장한다.
- 예외 흐름: 이름 중복 시 오류 메시지를 반환한다.

### UC-05: 할일 등록

- 액터: 인증된 사용자
- 사전 조건: 사용자가 로그인 상태이고 최소 하나의 카테고리가 존재해야 한다.
- 기본 흐름:
  1. 사용자가 할일 제목, 설명(선택), 종료예정일(선택), 카테고리를 입력한다.
  2. 시스템이 필수값(제목, 카테고리) 및 종료예정일 유효성을 검증한다.
  3. 시스템이 할일을 사용자 소유로 저장한다.
- 예외 흐름: 필수값 누락 또는 유효하지 않은 종료예정일 입력 시 오류 메시지를 반환한다.

### UC-06: 할일 목록 조회 및 필터링

- 액터: 인증된 사용자
- 사전 조건: 사용자가 로그인 상태여야 한다.
- 기본 흐름:
  1. 시스템이 로그인 사용자 소유의 할일 목록 전체를 표시한다.
  2. 사용자가 카테고리, 기간 종료 여부, 완료 여부 필터를 선택한다.
  3. 시스템이 선택된 필터 조건(AND)을 적용하여 해당 할일 목록을 반환한다.

### UC-07: 할일 수정

- 액터: 인증된 사용자
- 사전 조건: 수정 대상 할일이 로그인 사용자 소유여야 한다.
- 기본 흐름:
  1. 사용자가 할일의 제목, 설명, 종료예정일, 카테고리를 수정한다.
  2. 시스템이 유효성을 검증하고 변경 사항을 저장한다.
- 예외 흐름: 타인 소유 할일 접근 시 접근 거부 오류를 반환한다.

### UC-08: 할일 완료 처리

- 액터: 인증된 사용자
- 사전 조건: 대상 할일이 로그인 사용자 소유의 미완료 상태여야 한다.
- 기본 흐름:
  1. 사용자가 할일을 완료 처리한다.
  2. 시스템이 is_completed를 true로, completed_at을 현재 일시로 저장한다.
- 대안 흐름: 완료된 할일을 미완료로 되돌리면 is_completed를 false로, completed_at을 null로 초기화한다.

### UC-09: 할일 삭제

- 액터: 인증된 사용자
- 사전 조건: 삭제 대상 할일이 로그인 사용자 소유여야 한다.
- 기본 흐름:
  1. 사용자가 할일 삭제를 요청한다.
  2. 시스템이 해당 할일을 영구 삭제한다.
- 예외 흐름: 타인 소유 할일 접근 시 접근 거부 오류를 반환한다.

## 6. 도메인 모델 다이어그램 (텍스트 기반)

### 6.1 엔티티 관계도

```
+------------------+          +---------------------+
|      User        |          |      Category        |
+------------------+          +---------------------+
| PK id            |1        *| PK id               |
| email (unique)   |----------| FK user_id (Null=기본)|
| password (hash)  |          | name                |
| name             |          | is_default          |
| created_at       |          | created_at          |
| updated_at       |          +---------------------+
+------------------+                    |
         |                              | 1
         | 1                            |
         |                             * |
         |          +------------------+
         +----------+       Todo        |
                  * +------------------+
                    | PK id            |
                    | FK user_id       |
                    | FK category_id   |
                    | title            |
                    | description      |
                    | due_date         |
                    | is_completed     |
                    | completed_at     |
                    | created_at       |
                    | updated_at       |
                    +------------------+
```

### 6.2 사용자 인증 흐름

```
[비인증 사용자]
      |
      |-- 회원가입 --> [User 등록] --> [로그인 화면]
      |
      |-- 로그인 --> [Credentials 검증] --> [인증 토큰 발급]
                                                  |
                                          [인증된 사용자]
                                                  |
                              +---------+---------+---------+
                              |         |         |         |
                        [개인정보    [카테고리  [할일      [할일
                         수정]       관리]     등록]      조회/
                                                          필터링]
```

### 6.3 할일 상태 전이

```
[할일 등록]
     |
     v
[미완료 (is_completed: false)]
     |                  ^
     | 완료 처리         | 완료 취소
     v                  |
[완료 (is_completed: true, completed_at: 기록)]
     |
     v
[삭제 (영구 제거)]
```

### 6.4 필터링 조건 조합

```
할일 목록
    |
    +-- [카테고리 필터]      : 선택한 카테고리에 속한 할일
    |
    +-- [기간 종료 여부 필터]: due_date < today (기간 초과)
    |                         due_date >= today (기간 내)
    |
    +-- [완료 여부 필터]     : is_completed = true (완료)
                               is_completed = false (미완료)

    복수 조건 적용: 조건 A AND 조건 B AND 조건 C
```
