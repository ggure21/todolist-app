## 수정된 버그 (테스트 중 발견 및 수정)

### BUG-FIX-01: `useTodos` Zustand 셀렉터 무한 루프

- **증상**: 로그인 후 메인 페이지(`/`)가 완전 빈 화면. 콘솔에 `Maximum update depth exceeded` 오류 및 `The result of getSnapshot should be cached to avoid an infinite loop` 경고 발생
- **원인**: `frontend/src/features/todo/useTodos.ts`의 Zustand 셀렉터가 매 렌더마다 새 객체 참조를 반환해 React의 `useSyncExternalStore`가 무한 리렌더 루프에 빠짐
- **수정**: `useShallow` 적용으로 얕은 비교 기반 메모이제이션 적용
- **파일**: `frontend/src/features/todo/useTodos.ts`

```ts
// 수정 전
const filters = useFilterStore((s) => ({
  category_id: s.category_id,
  is_completed: s.is_completed,
  overdue: s.overdue,
}));

// 수정 후
const filters = useFilterStore(
  useShallow((s) => ({
    category_id: s.category_id,
    is_completed: s.is_completed,
    overdue: s.overdue,
  }))
);
```

## 발견된 버그 목록

### ❌ BUG-01: 카테고리 중복 생성 허용 (BR-C-04 위반)

- **시나리오**: B-1 / E-B1-1
- **재현 방법**: 기본 카테고리와 동일한 이름("업무")으로 새 카테고리 추가
- **기대 동작**: 중복 오류 메시지 표시, 카테고리 생성 거부
- **실제 동작**: 중복 카테고리가 정상 생성됨. 사이드바와 필터 드롭다운에 "업무"가 2개 표시됨
- **영향**: 카테고리 관리 혼란, 필터 기능 오동작 가능
- **스크린샷**: `screenshots/B1-BUG-duplicate-category-allowed.png`

### ❌ BUG-02: 로그인 실패 시 오류 메시지 미표시

- **시나리오**: B-3 / E-B3-1
- **재현 방법**: 올바른 이메일 + 잘못된 비밀번호로 로그인 시도
- **기대 동작**: "이메일 또는 비밀번호가 올바르지 않습니다." 등 단일 오류 메시지 표시
- **실제 동작**: 오류 메시지 없이 폼만 초기화됨 (서버는 400/401 반환하지만 UI 처리 누락)
- **영향**: 사용자가 로그인 실패 원인을 알 수 없음 (UX 심각)
- **스크린샷**: `screenshots/B3-BUG-login-error-no-message.png`

### ⚠️ 관찰 사항: 인증 토큰 비영속성

- **현상**: Zustand 인메모리 저장으로 인해 페이지 새로고침(F5) 시 로그아웃 상태로 초기화됨
- **시나리오 연관**: B-3 (디바이스 전환 후 데이터 동기화) — DB 데이터는 유지되나, 토큰이 만료되어 재로그인 필요
- **조치 권고**: `zustand/middleware`의 `persist` 미들웨어를 활용해 `localStorage` 에 토큰 저장 고려
